Vault-AI: Detailed Step-by-Step User Guide

Welcome to Vault-AI. This comprehensive guide will walk you through every step of using the AI-native Version Control System, from initial installation to advanced sandboxing.

Stage 1: Setup & Initialization

Step 1.1: Install Vault-AI

Ensure you have Vault-AI installed globally. Follow the instructions on the download page for your specific operating system (macOS/Linux via Homebrew, or Windows via PowerShell).

Step 1.2: Configure your AI Copilot

Before saving any code, you need to configure the AI engine that will write your commit messages.

Open your terminal and run:

vault-ai setup
  • You will be prompted to select an AI Engine: ollama (local), gemini, or openai.
  • If you choose a remote API (Gemini/OpenAI), paste your API key when prompted.
  • Vault-AI securely stores this key locally in your configuration file.

Step 1.3: Initialize your repository

Navigate to your project directory in the terminal and start tracking your code:

vault-ai init

This creates a hidden .vault folder inside your project. This is where Vault-AI will store all your secure snapshots and configuration data.

Stage 2: Writing Code and Saving Iterations

Step 2.1: Write your code normally

Make changes, add new files, delete old files, or refactor exactly as you normally would in your IDE.

Step 2.2: Let AI generate your commit message

Unlike Git, you do not need to stage files (e.g., no git add .). Simply ask Vault-AI to save your current state:

vault-ai save

How it works behind the scenes:

  1. Secret Guard: Vault-AI scans your code to ensure you didn't accidentally hardcode any sensitive API keys.
  2. Diff Analysis: The AI reads the differences between your code now and your last save.
  3. Generation: The AI generates a semantic, descriptive commit message explaining why you made the changes.

You will be asked to confirm, edit, or reject the AI's suggested message. Once you press 'y', your code is permanently saved to the vault timeline.

Step 2.3: Manual Override (Optional)

If you prefer to write your own message, you can bypass the AI by providing the message inline:

vault-ai save -m "Fixed the login button styling"

Stage 3: Reviewing Project History

Step 3.1: View current status

To see what files you have modified since your last save, run:

vault-ai status

Step 3.2: View the timeline

To review your past saves, AI-generated messages, and snapshot hashes, run:

vault-ai history

Stage 4: Advanced usage - Virtual Sandboxes

Sandboxes allow you to experiment. Think of them as isolated testing environments. If you want to try a massive refactor or new feature, do it in a sandbox.

Step 4.1: Enter a Sandbox

Create a fresh, isolated workspace identical to your current project:

vault-ai sandbox enter
You will be teleported into a new directory starting with sb_. Inside here, you can break the code, install new dependencies, or run tests without affecting your main project.

Step 4.2: Merge or Discard

If your experiment is successful, you can pull the updated code back into your main vault automatically:

vault-ai sandbox merge

If the experiment failed and you hate the code, simply delete the sandbox folder. Your main project remains untouched.

Conclusion

By relying on Vault-AI, you bypass the friction of staging, avoid committing secrets, and maintain an immaculate commit history written by artificial intelligence.
